# 专网是什么，以及如何在专网部署本系统（浏览器版优先）

本文面向答辩与落地说明：**先以浏览器访问的 Web 工作台为主**（`web/index.html` + `web/app.js`，由 `api_server.py` 提供页面与 API），再说明其与**专网**的关系与部署要点。

---

## 1. 什么是「专网」

**专网**（公安信息网、政务外网、行业生产网等）一般指与**公共互联网逻辑或物理隔离**、仅供授权单位与终端访问的网络域。典型特征包括：

- **访问控制**：终端准入（证书、802.1x、零信任客户端等）、账号与权限分级、操作审计。
- **数据边界**：业务数据、警情、音视频、布控类信息默认不出专网；对外交换走**安全交换区 / 网闸 / 脱敏网关**。
- **服务发现**：往往没有公网 DNS，使用内网域名或固定 IP；HTTPS 证书多为**内网 CA** 签发。
- **出站限制**：专网内服务器可能**不能直接访问**公网大模型 API，需通过**内网代理、私有化模型、或经审批的出网通道**。

本项目的「专网能力」在实现上体现为：前端通过同源或内网网关调用 **`/internal/v1/*`**（ASR、Canonical 警情、知识库等），并可配置 **`INTERNAL_API_TOKEN`** 与网关对齐。

---

## 2. 为什么先做「浏览器版」

- **交付形态清晰**：民警/研判员使用习惯以浏览器为主，无需安装客户端即可演示完整闭环（接警 → 研判 → 复核 → 布控辅助 → 性能与评审维度）。
- **与专网策略一致**：浏览器只访问**专网内可达的站点与接口**；敏感能力放在内网网关之后，由运维统一做 TLS、鉴权与审计。
- **迭代成本低**：静态资源 + Flask 单进程即可联调；后续若要换 Nginx + Gunicorn 或容器化，不改变业务接口契约。

## 3. 在专网里「做这个网站」需要哪些组件

### 3.1 最小运行面（演示 / 内网试点）

| 组件 | 作用 |
|------|------|
| **Python 运行环境与依赖** | 安装 `requirements.txt`（以当前仓库为准）。 |
| **`api_server.py`** | 提供 `/` 首页、`/web/*` 静态资源、`/api/*` 业务接口、`/uploads/*` 上传文件、`/internal/v1/*` 内部适配层。 |
| **大模型与向量库** | 研判依赖 LLM 与 RAG（如 Chroma）；专网内需改为**内网可访问的模型 endpoint**或**本地模型服务**。 |
| **（可选）ASR / 视频 / 知识库网关** | 对应 `INTERNAL_API_BASE` 等环境变量；若启用 Bearer 鉴权，由**网关**或运维向浏览器写入 `localStorage` 键 `plice_internal_token`（工作台不提供令牌输入框）。 |

### 3.2 推荐部署拓扑（文字版）

1. **应用服务器**（专网内）：部署本仓库代码，监听内网地址（如 `10.x.x.x:8000`）。生产建议前面加 **Nginx**，做 HTTPS 终止、限流与静态文件缓存。
2. **反向代理与 TLS**：使用内网证书；仅开放必要端口；管理面 SSH 与业务面分离。
3. **模型与检索**：与业务网同域或经策略允许的网段；**禁止**把警情原文未经脱敏直接发往公网 API。
4. **客户端**：专网办公终端浏览器访问 `https://警擎内网域名/`；书签与 DNS 使用内网解析。

### 3.3 环境变量与配置要点（摘要）

- **`.env`**：API Key、模型 Base URL、向量库路径等（参见仓库 `env.example`）。
- **专网网关**：`INTERNAL_API_BASE`、`INTERNAL_API_TOKEN`（服务端）；浏览器侧如需 Bearer，可自行写入 `localStorage.plice_internal_token` 或由网关注入请求头。
- **主机绑定**：开发常用 `0.0.0.0:8000`；专网生产应绑定内网网卡并配合防火墙白名单。

### 3.4 安全与合规提醒（答辩可一句话带过）

- 演示数据应**虚构或脱敏**；上传音视频仅存于本服务配置的 `data/uploads` 目录，由运维制定**保留周期与备份策略**。
- 日志中避免打印完整警情与令牌；内网接口应**鉴权 + 限频**，与公网完全隔离。

---

## 4. 启动命令（浏览器版）

在项目根目录：

```bash
python api_server.py
```

默认监听 `http://0.0.0.0:8000/`；专网内需改为内网 IP 或域名，并由反向代理提供 HTTPS。

---

## 5. 与代码的对应关系

- **页面与交互**：`web/index.html`、`web/app.js`、`web/styles.css`
- **HTTP 入口**：`api_server.py`
- **评审维度数据**：已并入 `GET /api/performance` 响应中的 `scorecard` 字段（与「性能与成效」页一次刷新同源）。

若后续需要移动端或独立客户端，可继续以同一套 `/api/*` 与 `/internal/v1/*` 契约为准进行扩展。
