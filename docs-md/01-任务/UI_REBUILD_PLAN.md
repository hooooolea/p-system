# UI 重构计划（简洁政务风）

## 目标

将 HTML 工作台（`web/`）的界面整体重构为**简洁、稳定、可读**的政务系统风格，满足以下硬要求：

- 不使用渐变背景或科技风视觉元素
- 不使用 emoji（标题、按钮、标签、提示、状态文案全部去除）
- 保留现有业务功能与数据流（研判、布控、导出、历史、复核、统计）

---

## 设计方向

- 视觉基调：浅色中性（白 + 灰 + 单一主色）
- 色彩策略：仅保留功能色（高危红、警示橙、提示蓝、成功绿）
- 布局策略：规整网格 + 卡片化信息区 + 清晰分区标题
- 文字策略：政务化文案，避免口语化和装饰性符号

---

## 执行步骤

### 第 1 步：全局样式重置

- [x] 去除渐变背景、光晕、阴影特效、复杂动效
- [x] 建立基础 CSS token（主色、边框色、文本色、背景色）
- [x] 保留轻量 hover 反馈（不使用炫技动画）

### 第 2 步：文案去 emoji + 命名统一

- [x] 页面标题、副标题、标签页、按钮、提示、状态文本全部去 emoji
- [x] 统一术语（例如“接处警研判”“目标布控”“数据概览”）

### 第 3 步：模块结构重排（不改逻辑）

- [x] 顶部：标题 + 状态条
- [x] 中部：Tab1（研判）/Tab2（布控）/Tab3（概览）
- [x] 结果区：摘要、关键信息、建议、依据、导出、复核
- [x] 侧栏：配置、统计、历史

### 第 4 步：组件外观统一

- [x] 卡片统一边框、圆角、内边距
- [x] 按钮层级统一：主按钮（primary）/次按钮（secondary）
- [x] 提示框统一语气：错误、警告、信息、成功

### 第 5 步：验证与回归

- [x] 语法检查：`python -m py_compile api_server.py`（及关键 `src/` 模块）
- [x] Lint 检查：`ReadLints`
- 手工验证：
  - 示例填充、研判流程、布控流程
  - 导出功能（Markdown/JSON）
  - 历史回填、复核记录导出
  - 演示模式开关可正常使用（若保留）

---

## 验收标准

- 界面中不出现任何 emoji
- 界面中不出现渐变背景或科技风装饰
- 全流程功能可用且无新增报错
- 视觉一致、信息层级清晰、适合正式展示
