# 布局改造步骤（基于政务/警务系统参考）

## 目标

把当前 HTML 工作台（`web/index.html` + `app.js`）的信息结构改为“政务系统可读布局”，强调：
- 任务导向（先看状态，再执行，再复核）
- 信息分层（总览指标 > 详情卡片 > 原始数据）
- 稳定操作路径（输入 -> 分析 -> 处置/复核 -> 导出）

---

## 外部参考结论（用于指导布局）

### 1) 警务平台常见结构
- 核心是“三块式”：
  - 左侧：列表/待办/历史
  - 中间：核心处置与地图/主工作区
  - 右侧：状态、预警、补充信息
- 强调“流程可视化、状态分段、处置节点跟踪”。

参考：
- [智慧警务全融合指挥平台研究（腾讯云）](https://developer.cloud.tencent.com/news/413865)
- [同题转载（安全内参）](https://www.secrss.com/articles/12843)

### 2) 政务系统常见结构
- 顶部必须有“统一状态区”（系统状态、时间、环境）
- 页面按任务分区，不混排
- 看板优先“关键信息 + 趋势”，少装饰、多可解释

参考：
- [政务设计系统综述（Datale）](https://www.datale.cn/tools/government-design-system)
- [数据看板设计方法（优设）](https://www.uisdc.com/dashboard-design-guide)

### 3) 成熟政府设计系统原则
- 明确可访问性、可读性、一致性
- 组件与文案规范化，而不是“自由发挥”

参考：
- [USWDS 设计系统](https://designsystem.digital.gov/)
- [GOV.UK Design System Patterns](https://design-system.service.gov.uk/patterns)

## 改造步骤（先做布局，再做样式）

## 无侧边栏方案（新增）

> 决策：去掉侧边栏，采用“顶部控制区 + 主体工作区 + 底部辅助区”的单主栏布局。  
> 目的：减少视觉分裂，保证演示时焦点集中在主流程。

### 信息迁移原则（侧边栏 -> 主界面）

- 原“系统设置（RAG开关、原始输出开关）”迁移到**顶部控制条**（与状态信息同区）。
- 原“会话统计”迁移到**Tab3 数据概览顶部**。
- 原“最近研判历史”迁移到**Tab1 输入区下方**，作为“历史回填区”。
- 原侧栏说明文案迁移到**页脚折叠说明区**（可选）。

### 新布局草图（无侧栏）

1. **顶部区（全局）**
   - 标题
   - 状态条（模型平台、RAG状态、当前时间）
   - 控制条（RAG开关、原始输出开关）

2. **中部区（标签页）**
   - Tab1：接处警研判（输入 -> 结果 -> 复核 -> 导出）
   - Tab2：目标布控（输入 -> 布控结构化结果 -> 缺失信息 -> 导出）
   - Tab3：数据概览（指标 + 图表 + 待复核列表）

3. **底部区（辅助）**
   - 常见问题折叠区（可选）
   - 版本信息/运行信息（可选）

### 无侧栏交互约束

- 每个 Tab 的“关键开关”不超过 2 个，避免顶部控制过载。
- 历史回填仅在 Tab1 展示，避免在全局出现重复入口。
- 统计信息以 Tab3 为唯一真值来源，避免多处显示不一致。

## 第 1 阶段：信息架构重排（不动业务逻辑）

- [x] 顶部改为“状态总览条”：模型平台、RAG状态、最近一次耗时
- [x] 去掉侧边栏，改为“顶部控制条 + Tab内功能区”
- [x] 侧栏设置迁移：RAG/原始输出开关移到顶部控制条
- [x] 侧栏历史迁移：最近10条历史移到 Tab1 输入区下方
- [ ] 侧栏统计迁移：会话统计移到 Tab3 顶部（当前在顶部控制区）
- [ ] Tab1 结果区固定为：
  - 总览指标（类型/风险/耗时/RAG）
  - 摘要与关键信息
  - 建议与法律依据
  - 复核与导出
- [ ] Tab2 固定为：
  - 输入区
  - 布控结构化输出
  - 缺失信息与人工复核
  - 导出

## 第 2 阶段：交互流程标准化

- [ ] 所有“生成”动作统一 3 段进度（检索/推理/解析）
- [ ] 所有异常统一“用户可读提示 + 原始错误折叠”
- [ ] 所有结果统一可导出（JSON + Markdown）
- [ ] 历史记录统一支持“一键回填重跑”

## 第 3 阶段：数据概览页重构

- [ ] 上方 4 个核心指标（总量、紧急、高风险、中风险）
- [ ] 中部风险分布柱图（固定色、固定排序）
- [ ] 下方“最近失败样例/待复核样例”列表（后续可接日志）

## 第 4 阶段：验收标准

- [ ] 首次打开 10 秒内可找到主操作入口
- [ ] 同一任务的操作路径不超过 3 次点击
- [ ] 所有页面都可“回看、复核、导出”
- [ ] 无侧边栏后，核心信息不丢失（设置/历史/统计都可达）
- [ ] 无花哨装饰，界面风格一致，适合正式汇报
